# 微算官网高端站开发计划

## 一、目标与约束

- **目标**：全球最高端商业官网水准，参考苹果官网，简洁明快、高大上；全面介绍微算（数据不出域的微型算力中心）。
- **约束**：仅网页端（PC）；无后台、无广告/商城/付费咨询；游客可用全部功能；登录/注册基于 Netlify；后端仅用 Netlify（Functions + Neon DB）；用户反馈不支持文件上传；数据已在 Neon，需分页+排序+筛选（来源、时间、关键词）。

---

## 二、技术栈（SEO 与规范）


| 层级    | 选型                            | 说明                                                                                                                                                  |
| ----- | ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 框架    | Next.js 16 App Router         | SSR/SSG 友好，metadata、sitemap、语义化 HTML                                                                                                                |
| 样式    | Tailwind CSS 4（现有）            | 简洁、可维护，支持深色/浅色与动效                                                                                                                                   |
| 部署    | Netlify                       | 使用现有 [netlify.toml](netlify.toml)：`publish = ".next"`，需确认与 [Next.js on Netlify](https://docs.netlify.com/frameworks/next-js/overview/) 推荐 output 一致 |
| 数据库   | Netlify DB（@netlify/neon）     | 在 serverless 中通过 `neon()` 访问，无需手写连接串；按 Netlify 规范使用 [Netlify DB](https://docs.netlify.com/build/data-and-storage/netlify-db/)                       |
| 身份    | Netlify Identity（gotrue-js）   | 仅邮箱/密码注册与登录，无第三方；注意：Netlify 已标注 Identity 为 deprecated，新项目可后续评估替代方案                                                                                  |
| 表单    | Netlify Forms + 自定义反馈 UI      | 反馈提交可继续用 [public/__forms.html](public/__forms.html) 或改为 Serverless + Neon 落库（便于与“数据表”统一）                                                            |
| 多语言   | Next.js [locale] + middleware | 路径形式：`/zh/...`、`/en/...`，默认中文；middleware 做 locale 检测与重定向                                                                                            |
| AI 客服 | Netlify Serverless Function   | 多 API 键轮询/并行与 fallback，见下文                                                                                                                          |


所有 API Key（含 AI）仅存放在 Netlify 环境变量中，不写入代码。

## 四、数据与 Neon 设计

- **Netlify DB（Neon）**：按规范安装 `@netlify/neon`，在 `netlify dev` / `netlify build` 时自动关联；在 Functions 内使用 `const sql = neon();` 访问。
- **反馈表**（若存库）：`feedback(id, name, email, message, created_at, optional user_id)`；可选与 Netlify Identity 的 user 关联。
- **“数据”页用表**：与需求“按来源、时间、关键词过滤”对应，例如：  
`content_items(id, source, created_at, keywords, title, summary, ...)`。  
数据您已存在 Neon，只需在该表上做分页、排序、筛选的 API（见下）。
- **迁移**：在仓库中提供迁移脚本（或一次性 SQL），通过 `netlify env:get NETLIFY_DATABASE_URL` 在本地/CI 执行，创建上述表结构。

## 五、Serverless 与 API 设计

- **数据列表 API**（供 `/[locale]/data` 使用）  
  - 路径：如 `GET /.netlify/functions/data` 或自定义 path（如 `/api/data`）。  
  - 查询参数：`source`、`keyword`、`from`、`to`（时间）、`sort`、`order`、`page`、`pageSize`。  
  - 在 Function 内用 `@netlify/neon` 查询并返回 JSON；实现分页（limit/offset 或 cursor）、排序、筛选。
- **用户反馈提交**（若用 Neon 而非仅 Netlify Forms）  
  - `POST /.netlify/functions/feedback`，写入 `feedback` 表；不处理文件上传。
- **AI 客服 API**  
  - 路径：如 `POST /.netlify/functions/ai-chat`。  
  - 请求体：`{ message: string, history?: [] }`。  
  - 逻辑：  
    - 从环境变量读取多个 AI API Key（DeepSeek、GLM、Moonshot、通义、腾讯、Spark、豆包、Anthropic、Gemini、DeepAI 等）。  
    - 实现“自动选择最快/可用”：例如并行请求多个 provider（带短超时），取最先成功返回的结果；或按优先级顺序 try/fallback。  
    - 系统提示词中注入：站点简介、主要栏目与路径（如 `/zh/product`、`/en/technology`），要求 AI 在回答时可给出“请前往 xxx 页面”并返回结构化链接（如 `{ url: '/zh/product', label: '产品页' }`）。
  - 响应：`{ reply: string, suggestedLinks?: { url, label }[] }`。  
  - 前端：全站悬浮按钮，打开聊天面板；支持中英文；可渲染建议链接为可点击跳转。

## 六、AI 客服前端与“导航到指定页面”

- 全站公共布局中增加“AI 客服”悬浮组件（如右下角图标）。
- 点击后展开对话面板：输入框、消息列表、加载状态。
- 调用 `POST /.netlify/functions/ai-chat`，若返回 `suggestedLinks`，在回复下方渲染为按钮/链接，点击即跳转到站内 `/[locale]/...`。
- 文案与 UI 支持中英文（与当前 locale 一致）。

## 七、身份与权限（Netlify Identity）

- 使用 **gotrue-js** 对接 Netlify Identity：  
  - 初始化：`APIUrl: 'https://<site>/.netlify/identity'`（或相对路径根据部署域名配置）。  
  - 提供：注册（邮箱+密码）、登录、登出、可选“忘记密码”。
- 不在页面上提供第三方登录；游客可访问所有页面与功能（数据页、反馈、AI 客服均不强制登录）。  
- 若反馈表需要“已登录用户”关联，则在提交时把 Identity 的 user id 写入 `feedback.user_id`；否则留空。

## 八、设计方向（Apple 级、简洁明快）

- **视觉**：大留白、大标题、高质量插图/产品图；主色与字体克制（如一套无衬线字体用于标题+正文）。  
- **动效**：首屏或区块进入时的轻量动画（如 fade-in、slide-up），避免花哨。  
- **排版**：清晰层级、合适行高与字距；表格页（数据、案例）保持可读性与对齐。  
- **响应式**：以 PC 为主，但保证平板/手机基本可用（导航、表格横向滚动或卡片化）。  
- **无障碍**：语义化 HTML、alt、焦点顺序、对比度符合 WCAG。

## 九、文件与目录建议（仅关键变更）

- **路由**：`app/[locale]/layout.jsx`（提供 locale、语言切换）、`app/[locale]/page.jsx`（首页）、`app/[locale]/product/page.jsx`、…、`app/[locale]/data/page.jsx`、`app/[locale]/feedback/page.jsx` 等。  
- **中间件**：`middleware.js` 中根据 Accept-Language / cookie 检测 locale，重定向到 `/zh` 或 `/en`。  
- **文案**：`locales/zh.json`、`locales/en.json`（或按模块拆分），页面通过 `useLocale`/context 或 getter 取文案。  
- **组件**：公共 `Header`/`Footer`（含语言切换、导航）、`AIChatWidget`、`DataTable`（筛选、排序、分页）、`FeedbackForm`。  
- **Serverless**：`netlify/functions/data.mts`、`netlify/functions/feedback.mts`（可选）、`netlify/functions/ai-chat.mts`；AI 键全部用 `Netlify.env.get('DEEPSEEK_API_KEY')` 等读取。  
- **迁移**：项目根或 `scripts/` 下提供创建 `feedback`、`content_items` 等表的 SQL/脚本，文档中说明用 `NETLIFY_DATABASE_URL` 执行。

## 十、实施顺序建议（不写排期与风险，仅逻辑顺序）

1. **基础与 i18n**：`[locale]` 路由、middleware、中英文文案结构、Header/Footer 与导航。
2. **内容页**：首页、产品、技术、解决方案、关于、案例、合作伙伴、联系我们（静态/SSG）。
3. **Neon + 数据页**：安装 `@netlify/neon`、迁移建表、数据列表 API、`/[locale]/data` 表格页（分页、排序、筛选）。
4. **反馈**：反馈表单页与提交（Netlify Forms 或 Serverless+Neon）。
5. **身份**：Netlify Identity 集成（注册/登录/登出），可选关联反馈。
6. **AI 客服**：Serverless 多 API fallback、系统提示词与站点导航、前端悬浮组件与建议链接跳转。
7. **SEO 与体验**：metadata、sitemap、robots、性能与无障碍检查。
8. **视觉与动效**：按 Apple 级与简洁明快做一轮统一调整。

---

## 十一、注意事项

- **API Key 安全**：所有 AI 及第三方 Key 仅放在 Netlify 环境变量中，不在代码与仓库中出现。  
- **Netlify Identity 状态**：当前仍可用，若后续迁移到 Auth0/Supabase 等，仅需替换 gotrue-js 调用与 JWT 校验处。  
- **发布目录**：若 Next.js 推荐使用 `output: 'standalone'` 或特定 export，需与 [netlify.toml](netlify.toml) 的 `publish` 及 Netlify 的 Next 构建文档对齐，避免部署失败。

按上述结构可实现：中英双语、表格数据页、分页/排序/筛选、用户反馈、AI 客服（含站内导航）、Netlify 身份与 Neon 数据，且符合“无自建后端、全部 Netlify、SEO 友好、高端官网”的 spec 要求。
